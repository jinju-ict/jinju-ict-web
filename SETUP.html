3첫 배포 절차

방법 A — GitHub 연동 (권장)

  • GitHub repo 생성 후 git push
  • Vercel 대시보드 → Add New Project → repo 선택
  • Framework 자동 감지 (Next.js) 확인 + 환경변수 4종 등록
  • Deploy → preview URL 확인 → main 푸시로 production

방법 B — Vercel CLI

pnpm i -g vercel
vercel link                          # 프로젝트 연결 (1회)
vercel env add RESEND_API_KEY        # 4종 env 등록
vercel env add CONTACT_INBOX_TO
vercel env add CONTACT_INBOX_FROM
vercel env add NEXT_PUBLIC_SITE_URL
vercel deploy                        # preview
vercel deploy --prod                 # production

4자주 보는 명령

pnpm install                         # 첫 설치
pnpm dev                             # 로컬 개발 (http://localhost:3000)
pnpm build                           # production build
pnpm start                           # production 서버 (build 이후)

pnpm lint                            # ESLint
pnpm typecheck                       # tsc --noEmit
pnpm check:security                  # 금지 단어 0건 검증
pnpm collect:assets                  # 포트폴리오 자산 재수집 (외부 폴더 변경 시)

5대표님이 직접 갱신할 영역 (placeholder)

AI 가 톤 맞춰 작성한 placeholder 부분 — 실 데이터로 교체 권장.

5-1. 팀 프로필 (src/lib/team.ts)

현재 Founder · Product / Founder · Engineering 2건 placeholder. 실명·실제 사진·구체 한 줄 소개로 교체 가능. 인원수 "N명" 표현 / 나이 노출은 사이트 룰상 금지 — "소수 정예" 톤 유지.
  • displayName — 실명 또는 닉네임
  • oneLiner — 강조하고 싶은 한 줄
  • strengths — 키워드 chip (현재는 일반화)
  • 아바타 사진 — 현재 이니셜 그라데이션. 실사진은 별도 컴포넌트 수정 필요

5-2. 사주 앱 메타 (src/lib/saju.ts)

appName 이 가제 ("사주"). 정식 브랜드명 확정 후 갱신. 출시 후엔 status 를 "준비중"에서 "출시" 등으로 변경, App Store / Play Store 다운로드 링크 추가 필요 (CTA 의 #contact 앵커도 다운로드 링크로 교체).

5-3. 도메인 + production URL

도메인 결정 후 NEXT_PUBLIC_SITE_URL 갱신 (Vercel env + .env.local). metadata / robots / sitemap / OG 모두 자동 반영.

5-4. Resend 도메인 검증

현재 발신 onboarding@resend.dev (테스트 도메인). 자체 도메인에서 발신하려면 Resend 대시보드 → Domains → Add 후 DNS TXT/MX 레코드 등록. 검증 완료 후 CONTACT_INBOX_FROM 을 noreply@<도메인> 으로 갱신.

5-5. 포트폴리오 자산 보강 (선택)

현재 owned 3건 + Andy 1건 = 일부만 실자산. 나머지는 의도된 placeholder (사내 도구·외부 회사 자산 보안 룰). owned 의 스크린샷 추가 원하면 해당 폴더에 logo.png / screenshot.png 등 두고 pnpm collect:assets 재실행.
